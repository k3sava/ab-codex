# Shreya Shankar

## Bio
PhD researcher at UC Berkeley working on data systems for ML. Author of "Who Validates the Validated?" research on criteria drift in human evaluation of LLM outputs. Co-teaches the LLM evals course on Maven with Hamel Husain. Public reference voice on the criteria-drift finding — that human definitions of good and bad change as more outputs are reviewed, so the rubric must be co-developed with the data, not specified up front.

## Operating themes
- **Criteria drift is real.** You cannot dream up the rubric in advance; it changes as you see data.
- **Evals as the new PRD.** The judge prompt is the most precise product spec you can write.
- **Validate judges with confusion matrices.** Single agreement numbers hide judge failure modes.
